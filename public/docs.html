API Docs

Integrate Nodetree into your workflow via a single POST endpoint.

Endpoint

POST /bfhl

Send an array of edge strings and receive a validated hierarchy response.

Request

{
  "data": ["A->B", "A->C", "B->D", "C->E"]
}

Each item in data should be a string in X->Y format where X and Y are uppercase letters.

Response

{
  "hierarchies": [
    {
      "root": "A",
      "depth": 2,
      "has_cycle": false,
      "tree": { "A": { "B": { "D": {} }, "C": { "E": {} } } }
    }
  ],
  "invalid_entries": [],
  "duplicate_edges": [],
  "summary": {
    "total_trees": 1,
    "total_cycles": 0,
    "largest_tree_root": "A"
  }
}

Validation Rules

1
Format

Only [A-Z]->[A-Z] edges are accepted. Self-loops rejected.

2
Deduplication

Duplicate edges are kept once; extras listed in duplicate_edges.

3
Single Parent

A node can have at most one parent. First-encountered parent wins.

4
Cycles

Detected via DFS colouring. Flagged with has_cycle: true.

Try it with cURL

curl -X POST https://your-domain.com/bfhl \
  -H "Content-Type: application/json" \
  -d '{"data": ["A->B", "A->C", "B->D"]}'
